# FinPay - Distributed Payment Infrastructure

## Project Purpose
Learning platform to deeply understand real backend/distributed systems concepts used in production payment systems (Paytm Wallet, PhonePe style).

## Current State
- Early stage - has README.md and architecture diagram
- Goal is NOT a startup/product but learning backend engineering

## Key Focus Areas
- Distributed systems patterns (Saga, distributed locking)
- Event-driven architecture
- Payment flow engineering
- Redis in production (queues, streams, locks)
- MongoDB for financial data
- BullMQ for async processing
- Security (JWT, rate limiting, double-spend prevention)
